Section 2: Interactive Answers & Keys Config

Calculation workflow

Base path: equation given in the prompt (substitute) or recall from equation sheet. Optional steps (each +1 mark when ticked): rearrangement, unit conversion, significant figures.

Equation select step

Student chooses the correct equation from the sheet (or a dropdown). Not marked — informational only.

Substitution step

Structured mode shows one input per quantity in the equation. Free text uses a single line (legacy).

Unit conversion step

Example: question gives 1.2 km — step label “Convert 1.2 km to metres”, correct converted value 1200, tolerance 0.001 (student enters 1200 ± 0.001).

Rearrangement step

Student picks the correctly rearranged formula (or numeric rearrangement with values). Can be added to either equation-given or equation-recall questions.

Significant figures step

Final answer must be given to this number of significant figures (checked on the calculate input).

Final answer step

Correct value and tolerance are set in the answer key fields above. Optional message if the final number is wrong.

Wrong answer walkthrough (optional)

Shown step-by-step when the final answer is incorrect. Add one row per teaching step.

Optional custom feedback for each step is set under Feedback if wrong in the step panels above. Built-in defaults are used if left blank.

Note: If you leave the "Section 3: Step-by-Step Mark Checkpoints" completely blank below, this question will grade automatically using basic Required/Optional keyword algorithms.

Pool marking: the student earns the marks-per-item value for each distinct acceptable answer they name, capped at the question's max marks. Ideal for "State two…", "Name three…" style questions. Set AO1 = max marks in the metadata below. Leave Section 3 checkpoints blank.

Section 3: Step-by-Step Mark Checkpoints

Short Answer Questions: Each matched row awards 1 mark (AO-specific). If left blank, grading uses the required/optional keywords in Section 2 instead. Optional feedback diagrams appear when a student misses that checkpoint.
Keyword syntax: comma , = all parts required (AND); pipe | = synonyms (OR). Example for one mark: increase, potential difference|voltage|volts needs both “increase” and one of the voltage terms.
Numeric Questions: Feedback is configured in Section 2 (calculation workflow panels) — this section is not used.
MCQ: Add one generic remediation row (feedback field only). Shown below any per-option feedback from Section 2 when a student picks a wrong answer.

Import Packaged CSV / TSV Lists

Bulk-import MCQ and short-text questions from Google Sheets or Excel. Use the tab-separated header template below as row 1 in your sheet. Empty subject, paper, or tier cells inherit the fallback values configured here. Legacy comma-separated positional files (24 columns, no header) are still supported.

Column template & field guide

Delimiter: Tab (.tsv) recommended for Google Sheets. Comma (.csv) also works when fields are quoted.

Keywords: keywords_required / keywords_optional — comma separates mark groups; pipe | = synonyms within a group (e.g. gravity|gravitational, mass).

Pick N pool (partial credit): keyword_pool — comma-separated acceptable answers (| = synonyms), e.g. coal, oil|petroleum, gas, nuclear|uranium. Awards pool_marks_per_hit (default 1) per distinct answer, capped at max_marks. Ideal for "State two…". When set, keyword/Section 3 fields are ignored.

Hints: Separate multiple hints with ||| (revealed one at a time in practice).

MCQ feedback: mcq_feedback_a…d map to option_a…d text (shown when that wrong option is chosen).

Section 3: mp1_ao, mp1_keywords, mp1_feedback, mp1_image_url … up to mp6. When any mark-point row is present, grading uses checkpoints (Section 2 keywords are ignored).

Images: image_url = question stem diagram; mpN_image_url = feedback diagram when that checkpoint is missed (public HTTPS URL).
